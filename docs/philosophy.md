# Философия и контекст

Сцена - это рабочий контейнер состояния мира. Она хранит набор сущностей и задает их жизненный цикл в кадре: когда запускать логику, когда делать fixed update, когда готовить данные к рендеру.

Сущность - это идентичность и базовые свойства. В `termin-scene` у сущности есть id, имя, флаги, transform, связи в иерархии и список компонентов.

Компоненты - это поведение и данные. Сущность получает функциональность не через наследование, а через набор компонентов: можно добавлять, снимать и комбинировать их по задаче.

Библиотека написана на C, потому что C - lingua franca системного мира. Такое ядро проще встроить в приложение на любом языке через FFI и держать общий контракт для C/C++/Python/C#-слоев.

Отличия этой реализации от типичного ECS: гибрид object-компонентов и SoA-архетипов, generational handles для безопасной валидности ссылок, и scene extensions как отдельный механизм расширения без изменения core.
